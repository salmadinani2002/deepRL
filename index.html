Dans le cadre du cours **Deep Reinforcement Learning**, nous avons étudié l'évolution des méthodes d'apprentissage par renforcement : ``` Value-Based (DQN) ──► Policy Gradient (REINFORCE) ──► Actor-Critic ──► PPO ✅ ``` | Méthode | Principe | Limite | |---|---|---| | **DQN** | Apprend Q(s,a) | Espaces discrets uniquement | | **REINFORCE** | Policy Gradient pur | Haute variance, instable | | **Actor-Critic** | Politique + Valeur | Peut diverger | | **PPO** ⭐ | AC + Clipping | **Standard industrie** | --- ### 🎭 Architecture Actor-Critic PPO repose sur **deux réseaux de neurones** qui coopèrent : ``` ┌─────────────────────────────────────────┐ │ AGENT PPO │ │ │ État s_t ────────►│ ┌─────────┐ ┌─────────────┐ │ │ │ │─────►│ ACTOR │─────┼──► Action a_t │ │ Couches │ │ π_θ(a|s) │ │ (distribution) │ │Partagées│ └─────────────┘ │ │ │ │ ┌─────────────┐ │ │ │ │─────►│ CRITIC │─────┼──► Valeur V_φ(s_t) │ └─────────┘ │ V_φ(s) │ │ (scalaire) │ └─────────────┘ │ └─────────────────────────────────────────┘ ``` ### ⚡ Le Mécanisme de Clipping — Innovation Clé de PPO L'innovation majeure de PPO est sa **fonction objectif tronquée** qui empêche des mises à jour trop agressives. **Le ratio de probabilité :** $$r_t(\theta) = \frac{\pi_\theta(a_t \mid s_t)}{\pi_{\theta_{old}}(a_t \mid s_t)}$$ **La fonction objectif clippée :** $$L^{CLIP}(\theta) = \mathbb{E}_t \left[ \min\left( r_t(\theta) \hat{A}_t,\ \text{clip}(r_t(\theta), 1-\varepsilon, 1+\varepsilon) \hat{A}_t \right) \right]$$ ``` L^CLIP │ ╔══════╗ ← plafond (1+ε)·Â_t │ ┌─────╝ ║ │ │ ║──── zone clippée │────┤ └───────────────── │ │ │ └─── plancher (1-ε)·Â_t └──────────────────────────► r_t(θ) 1-ε 1 1+ε ``` > Le ratio `r_t(θ)` est **forcé de rester dans [1−ε, 1+ε]**, évitant que la politique change trop drastiquement en une étape. ### 🔄 Erreur TD et Avantage La communication entre Actor et Critic se fait via l'**erreur TD** : $$\delta_t = r_t + \gamma \cdot V_\phi(s_{t+1}) - V_\phi(s_t)$$ | Symbole | Signification | |---|---| | `r_t` | Récompense immédiate | | `γ` | Facteur d'actualisation (discount) | | `V_φ(s_{t+1})` | Valeur future estimée par le Critic | | `V_φ(s_t)` | Valeur de l'état actuel | | `δ_t > 0` | L'action était **meilleure** qu'attendu → augmenter sa probabilité | | `δ_t < 0` | L'action était **pire** qu'attendu → diminuer sa probabilité | --- ## 🏗️ Architecture ### Structure du Code ```python class ActorCritic(nn.Module): def __init__(self, obs_dim, action_dim): super().__init__() # Couches partagées self.shared = nn.Sequential( nn.Linear(obs_dim, 64), nn.Tanh(), nn.Linear(64, 64), nn.Tanh() ) # Tête Actor → distribution sur les actions self.actor_head = nn.Linear(64, action_dim) # Tête Critic → valeur scalaire de l'état self.critic_head = nn.Linear(64, 1) def forward(self, state): shared = self.shared(state) # Actor : probabilités softmax action_probs = F.softmax(self.actor_head(shared), dim=-1) # Critic : valeur scalaire state_value = self.critic_head(shared) return action_probs, state_value ``` ### Boucle d'Entraînement PPO ```python def update_ppo(memory, model, optimizer, epsilon=0.2, epochs=4): for _ in range(epochs): # Calcul des probabilités avec la politique actuelle action_probs, state_values = model(memory.states) # Ratio r_t(θ) = π_θ(a|s) / π_θ_old(a|s) ratios = action_probs / memory.old_probs # Avantage normalisé advantages = (memory.returns - state_values.detach()) advantages = (advantages - advantages.mean()) / (advantages.std() + 1e-8) # Fonction objectif clippée surr1 = ratios * advantages surr2 = torch.clamp(ratios, 1 - epsilon, 1 + epsilon) * advantages # Perte totale = Actor + Critic + Entropie actor_loss = -torch.min(surr1, surr2).mean() critic_loss = F.mse_loss(state_values.squeeze(), memory.returns) entropy = -(action_probs * action_probs.log()).sum(-1).mean() loss = actor_loss + 0.5 * critic_loss - 0.01 * entropy optimizer.zero_grad() loss.backward() optimizer.step() ``` --- ## 🔧
